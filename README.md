# JATLAS

**Jav Actress Tier Ledger & Asset System** 是一个面向 NAS + Emby 本地收藏架构的 AV 影片资产管理工具。

它解决的不是“如何下载更多内容”，而是另一个更现实的问题：当收藏规模不断膨胀、硬盘空间持续被占用、Emby 条目和真实文件逐渐脱节时，如何建立一套可持续的管理机制，让个人收藏保持可控、可查、可整理。

## 面对的问题

### 收藏很容易变成仓鼠症

很多 AV 收藏者一开始只是保留喜欢的作品，后来会逐渐进入一种“先存下来再说”的状态。演员、系列、厂牌、画质版本、字幕版本不断叠加，收藏从兴趣爱好变成持续扩张的文件堆。

真正的问题不是某一部影片占多少空间，而是缺少一套判断机制：

- 什么内容值得长期保留
- 什么内容只是短期兴趣
- 哪些演员已经明显超量
- 哪些分级需要收缩
- 哪些文件已经不再符合当前偏好

没有规则时，收藏会自然膨胀；没有台账时，清理只能靠印象；没有阈值时，硬盘报警往往才是第一次真正介入。

### 当代存储空间并不便宜

NAS 的容量看起来很大，但高质量影片会非常快地吞掉空间。硬盘、阵列、备份、机箱、电费和后续扩容都是真实成本。

如果没有管理机制，扩容会变成唯一答案：空间满了就加盘，加盘之后继续囤积，最后维护成本越来越高。对个人收藏来说，这不是一个健康的长期模型。

更好的方式是让空间使用和收藏价值挂钩：高价值内容可以占用更多空间，低价值内容应该被限制、审视或逐步清理。

### NAS 和 Emby 解决了播放，不解决治理

NAS 负责存储，Emby 负责媒体库和播放体验。它们很适合承载收藏，但并不会天然回答这些管理问题：

- 某个演员在本地到底占了多少部作品
- 这个数量是否已经超过自己设定的上限
- 当前文件夹结构和 Emby 人物 ID 是否一致
- 新增文件夹是否已经进入台账
- 最近的资产变化是否合理

Emby 是媒体库，不是收藏策略系统；NAS 是存储层，不是价值判断层。两者之间需要一个专门的管理层，把“文件在哪里”“Emby 识别成什么”“我是否值得继续保留”连接起来。

## JATLAS 的解决方案

JATLAS 在 NAS + Emby 之上建立一层本地资产台账。它把收藏拆成可管理的对象：演员、分级、视频数量、Emby ID、存储路径、资产日志和风险状态。

它的思路很简单：先建立规则，再同步事实，最后根据规则持续维护空间。

## 机制一：用分级替代模糊喜好

JATLAS 以 Tier 作为收藏分级。每个演员都必须归入一个明确层级，每个层级都可以设置视频数量上限。

这让“喜欢”从模糊感受变成可执行规则：

- 核心收藏可以拥有更高上限
- 普通收藏只能保留有限数量
- 边缘兴趣不应该无限增长
- 不同分级之间可以形成清晰的空间预算

分级的意义不是给演员贴标签，而是帮助用户承认资源有限，并把有限空间留给真正高价值的收藏。

## 机制二：用风险状态暴露失控点

当演员的视频数量接近或超过所属分级上限时，JATLAS 会把它标记为风险状态。

这使清理工作不再依赖“我记得好像很多”，而是变成明确的维护队列：

- 哪些演员已经超量
- 哪些演员接近上限
- 哪些分级整体压力较高
- 哪些新增内容会推高风险

风险状态的目标不是强制删除，而是让空间压力可见。只要压力可见，用户就能做出更理性的取舍。

## 机制三：和 Emby 对账，而不是重复维护

JATLAS 可以通过 Emby API 同步演员 ID 和视频数量。Emby 仍然负责媒体识别和播放体验，JATLAS 负责把这些事实转化为收藏管理信息。

这种分工避免了两套系统各自维护：

- Emby 中的人物 ID 可以写回 JATLAS
- Emby 统计到的视频数量可以更新台账
- JATLAS 根据数量和分级上限计算风险
- 用户在 JATLAS 中决定是否调整分级或整理资产

也就是说，Emby 提供媒体事实，JATLAS 提供管理判断。

## 机制四：扫描 NAS 路径，把文件夹纳入台账

很多收藏并不是先进入系统，再进入硬盘；现实往往是文件已经存在 NAS 上，台账反而是后补的。

JATLAS 支持为分级绑定存储路径，扫描路径下的演员文件夹，并将未建档的名称批量导入。

这让已有收藏可以逐步被纳入管理，而不是要求用户从零开始手动录入。NAS 文件夹不再只是存储结构，也可以成为台账建设的入口。

## 机制五：用日志理解资产变化

JATLAS 会记录演员创建、删除、更新和视频数量变化。仪表盘会把近期资产变动、分级分布和风险概况集中展示。

这让用户能看到收藏结构的变化趋势：

- 最近是否新增过多
- 哪些分级增长最快
- 哪些演员数量变化明显
- 当前台账是否仍然符合自己的收藏策略

长期来看，日志比单次清理更重要。它让收藏维护变成持续过程，而不是每次硬盘满了才临时救火。

## 为什么是桌面应用

JATLAS 是本地优先的 Electron 桌面应用，使用 SQLite 保存数据。

AV 收藏管理天然带有隐私属性，也不需要多人在线协作。桌面形态更适合这个场景：

- 数据保留在本机
- 不依赖公网服务器
- 更容易访问本地配置和 NAS 路径
- 适合自用或小范围分享
- 不需要维护一套 Web 部署环境

它更接近一个私人资产工作台，而不是面向公众的在线服务。

## 当前能力

JATLAS 当前桌面版已经覆盖主要维护流程：

- 登录认证
- Tier 分级管理
- 演员台账管理
- 分级上限与风险状态
- Dashboard 总览
- Emby ID 同步
- Emby 视频数量同步
- NAS 路径扫描
- 批量导入演员
- 长任务取消与重试
- 本地 SQLite 存储
- mac / Windows 桌面打包

这些能力共同构成一套“收藏治理层”：它不替代 NAS，也不替代 Emby，而是在两者之上补上规则、台账、风险和维护流程。

## 项目边界

JATLAS 不提供内容下载，不提供公开分发，不提供在线账号体系，也不试图成为媒体播放器。

它只关注一件事：帮助已经使用 NAS + Emby 管理本地 AV 收藏的人，把不断膨胀的影片资产重新纳入可理解、可控制、可持续维护的状态。
