# JATLAS

**Jav Actress Tier Ledger & Asset System** 是一个本地自用的 Electron 桌面应用，用来管理影视收藏中的演员分级、资产数量、Emby 数据联动和存储扫描导入。

它的核心目标不是做公开内容平台，而是把分散在 Emby、NAS 文件夹和个人判断里的收藏信息，整理成一个可长期维护、可快速判断风险、可批量更新的本地台账。

## 为什么需要 JATLAS

影视收藏增长到一定规模后，单靠文件夹命名、Emby 条目或临时表格很难回答几个关键问题：

- 哪些演员属于高优先级收藏，哪些只是普通保留
- 某个演员当前有多少资产，是否已经接近或超过设定上限
- Emby 中的演员 ID 和本地台账是否一致
- NAS 某个分级目录里新增了哪些演员文件夹
- 最近的增删改动对整体资产结构产生了什么影响

JATLAS 把这些问题收束到一个本地桌面工作台中，让收藏维护从“靠记忆和手工检查”变成“按分级、数量、同步状态和日志来判断”。

## 核心功能

### 演员分级台账

JATLAS 以 Tier 作为收藏层级，每个演员都归属于一个明确分级。分级可以设置资产上限，用于表达收藏优先级、容量规划和清理策略。

演员台账记录姓名、所属分级、视频数量和 Emby ID。它既是收藏索引，也是后续同步、扫描和统计的基础数据。

### 风险与容量判断

系统会根据分级上限和演员当前资产数量计算风险状态，让超量、接近上限或正常范围的演员更容易被识别。

这类信息的意义在于减少盲目扩充：用户可以优先处理真正需要关注的分级和演员，而不是在大量文件夹之间反复确认。

### Emby 联动

JATLAS 可以和 Emby 进行两类同步：

- 根据演员名称匹配并写入 Emby ID
- 根据 Emby 影视库统计演员对应的视频数量

同步任务采用可取消、可重试的长任务模型，适合处理较大的演员列表。它让本地台账能跟随 Emby 资产变化更新，避免台账和媒体库长期脱节。

### NAS 路径扫描与批量导入

每个分级可以对应实际存储路径。JATLAS 能扫描目录中的演员文件夹，并将尚未进入台账的名称批量导入。

这解决的是“先有文件、后补台账”的常见场景：当存储目录已经形成规模时，不需要逐个手动录入，扫描结果可以作为整理入口。

### 仪表盘与变更日志

桌面端提供总览数据、分级分布、风险概况和最近资产变动图表。资产日志记录演员创建、删除、更新和数量变化，帮助用户理解近期台账变化。

仪表盘的意义不是做复杂报表，而是让用户每次打开应用时都能迅速知道当前收藏结构是否健康、哪些地方需要处理。

### 本地优先的数据模式

JATLAS 使用 SQLite 作为桌面端数据存储，数据保存在本机。配置、数据库和运行日志都围绕本地桌面使用场景设计。

这种模式适合自用和小范围分享：不依赖服务器，不需要公网部署，也不会把私人收藏信息放到远程数据库。

## 当前形态

JATLAS 已从早期 Web 控制台迁移为桌面应用。当前版本以 Electron 客户端为主，包含认证、分级管理、演员管理、仪表盘、Emby 同步、存储扫描和批量导入等核心能力。

历史 Web 版本的视觉方向被保留下来，并适配为更适合桌面工作台的深色界面：左侧导航、顶部状态栏、紧凑表格、风险卡片和批量任务面板共同组成主要操作空间。

## 使用边界

JATLAS 面向本地收藏管理，不提供内容下载、公开分发、在线账号体系或多人协作平台能力。

它更像一个私人资产维护工具：帮助用户建立清晰的分级标准、减少重复检查、保持 Emby 与本地存储之间的数据一致性，并让长期收藏维护变得可追踪。
